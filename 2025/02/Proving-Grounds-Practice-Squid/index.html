Proving Grounds Practice Lab中Squid打靶过程记录

Difficulty

Easy

Scan

Foothold

  1. 搜索 squid 4.14 exploit, 找不到可以利用的expolit code
  2. 设置代理 192.168.212.189:3128, 再访问192.168.212.189 不通
  3. 设置代理后，重新nmap扫描端口，没有任何输出
  4. smb匿名登录失败，尝试administrator常见弱密码无法成功， 使用hydra 爆破smb没有结果

几个小时没有进展，查看hits，确实是需要通过http-proxy再次做信息收集

原来是export proxy这种方式nmap无法生效，使用nmap –proxies参数再次做信息收集， 还是没有收获，看提示是在8080端口，以后遇到相同的情况写脚本来扫？ curl --proxy http://ip:port http://ip:port

gobuster目录扫描， 发现phpsysinfo、adminer、phpmyadmin三个路径，浏览器设置代理后，也可以直接看到这三个页面的入口, 除此之外还有phpinfo

  1. 搜索wampserver 3.2.3版本漏洞，找到 https://www.exploit-db.com/exploits/50094， 测试之后发现无法利用。

  1. 进入 http://192.168.212.189:8080/phpsysinfo/index.php?disp=bootstrap, 展示系统信息, 系统是windows server 2019， 后面提权可能有用

  1. http://192.168.212.189:8080/adminer/ adminer 版本号4.7.7 搜索已知漏洞，无结果；尝试root/admin账号常见的弱密码无法登录，手工检测sql注入无结果。

  1. phpadmin http://192.168.212.189:8080/phpmyadmin/ 简单sql注入无结果，用户名root，无密码登录成功。

    先检查了一下user表，没有和系统用户有关的账号信息

检查下文件写入权限， <a data-lark-is-custom="true" href="http://192.168.212.189:8080/phpmyadmin/url.php?url=https://dev.mysql.com/doc/refman/5.5/en/select.html">SELECT</a>`` @@secure_file_priv 结果为空表示可以写入任务目录，为null不允许写入

接下来的思路就是写webshell了，需要先知道当前server的绝对路径

首页的phpinfo进入可以看到当前路径 C:/wamp/www/

接下来就写webshell， 反弹shell了

eacalation

官方walkthrough中，反弹shell之后得到的是local service的shell，offsec pg lab的有很多已经更新了，但是walkThrough没有更新，可以确认的是都变得更简单了。

提权方法记录如下:

  1. 查看权限，发现有些权限缺失了，通过文章 发现 local service or network service被配置为受限的权限 restricted set of privileges *, *完整的权限可以通过 scheduled task获取

kali 获取反弹shell

  1. 至此权限中还是没有 SeImpersonatePrivilege，但是可以通过创建一个 ScheduledTaskPrincipal 来获取, 可以在其中指定 seimpersonatePrivilege在必需的privilege属性中

kali监听新的4444端口， 发现已经有了 SeImpersonatePrivilege权限

然后就可以从 https://github.com/itm4n/PrintSpoofer下载PrintSpoofer.exe来提权了

  1. 基础工具使用还是不够熟练
